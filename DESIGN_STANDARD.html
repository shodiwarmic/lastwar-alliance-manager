Frontend Design Standard

Alliance Manager · Complements CLAUDE.md · Last updated May 2026

This is the canonical reference for all UI and CSS work. It covers the design system: tokens, theming, components, and CSS/JS conventions specific to the visual layer. Backend patterns, Go conventions, and template architecture are in CLAUDE.md.

Theme System

The app carries two theming mechanisms. Only the attribute system is canonical for new code.

Canonical: html[data-theme]

theme.js sets data-theme="light" or data-theme="dark" on <html> synchronously on page load, before first paint. All CSS written during and after the redesign uses this attribute.

/* ✅ Correct — new page CSS */
html[data-theme="dark"] .my-element {
    background: var(--color-surface);
}

Legacy: html.theme-* (do not use for new code)

The old class-based system (html.theme-dark, html.theme-light, html.theme-auto) remains in styles.css to support page CSS written before the redesign. Do not write new dark-mode overrides targeting these classes. theme.js sets both simultaneously; the canonical [data-theme] tokens always win due to specificity.

color-scheme property

When setting the native color-scheme hint on an element (e.g. a date picker), include both selectors:

/* ✅ Covers both systems */
html.theme-dark .my-input,
html.theme-auto .my-input,
html[data-theme="dark"] .my-input {
    color-scheme: dark;
}

Tab switching

The global rule sets .tab-content { display: none }. You must show the initial active tab explicitly on DOMContentLoaded, and always use style.display = 'block' — never style.display = ''.

document.addEventListener('DOMContentLoaded', () => {
    const activeBtn = document.querySelector('.tab-btn.active');
    if (activeBtn) {
        const tab = document.getElementById('tab-' + activeBtn.dataset.tab);
        if (tab) tab.style.display = 'block'; // NOT style.display = ''
    }
});

Component Patterns

Buttons

<!-- Standard weights -->
<button class="btn btn-primary">Save</button>
<button class="btn btn-secondary">Cancel</button>
<button class="btn btn-danger btn-sm">Delete</button>
<button class="btn btn-warning btn-sm">Flag</button>
<button class="btn btn-ghost btn-sm">Clear</button>

<!-- Icon + label (icon from the sprite) -->
<button class="btn btn-primary"><svg class="svg-icon" width="14" height="14" aria-hidden="true"><use href="/icons.svg#icon-device-floppy"/></svg> Save</button>
  • .btn-ghost — tertiary / low-weight actions (card "Edit", "Clear filter", "Today" nav, modal "Close"). Transparent with a --color-border border; lighter than .btn-secondary.
  • Do not use .primary-action-btn / .secondary-action-btn — deprecated.
  • .btn is display:inline-flex; gap:6px, so an SVG icon + text label align automatically.
  • .button-group — a display:flex; gap:10px container for a row of buttons. Use it instead of margin-right.
  • .btn-sm reduces padding for inline/table contexts.

Modals

Modals go in {{define "modals"}}, not {{define "content"}}. Placing them inside content traps them in the container's stacking context and breaks position:fixed backdrops.
{{define "modals"}}
<div id="my-modal" class="modal">
  <div class="modal-content">
    <h2>Title</h2>
    <!-- form content -->
    <div class="modal-actions">
      <button class="btn btn-primary" id="my-save">Save</button>
      <button class="btn btn-secondary" id="my-cancel">Cancel</button>
    </div>
  </div>
</div>
{{end}}
// Open
document.getElementById('my-modal').style.display = 'flex';
// Close
document.getElementById('my-modal').style.display = '';
// Backdrop click
modal.addEventListener('click', e => { if (e.target === modal) modal.style.display = ''; });
Never add class="hidden" to a .modal — it is already display:none by default. Adding .hidden (display:none !important) prevents style.display='flex' from taking effect.

No corner ×. Close via a Cancel/Close button in .modal-actions plus backdrop click. Do not add <span class="close">&times;</span>. Viewer-style modals (image/document preview) with no footer close on backdrop click alone.

Tabs

<div class="tab-bar">
  <button class="tab-btn active" data-tab="overview">Overview</button>
  <button class="tab-btn"       data-tab="history">History</button>
</div>
<div id="tab-overview" class="tab-content">...</div>
<div id="tab-history"  class="tab-content">...</div>
Non-canonical patterns — do not use. .tab-content.active (Rankings, Admin) and .tab-panel + .hidden (Train, Accountability) both bypass the global .tab-content { display:none } rule. Migration is deferred; new code uses .tab-content + style.display='block' only.

Data Tables

<div style="overflow-x:auto">
  <table class="data-table">
    <thead>
      <tr>
        <th>Name</th>
        <th>Rank</th>
        <th style="width:1%;white-space:nowrap">Actions</th>
      </tr>
    </thead>
    <tbody id="my-table-body"></tbody>
  </table>
</div>
  • Pin the actions column: width:1%; white-space:nowrap.
  • overflow-x:auto goes on the wrapper div, not the table.
  • Add a page-specific subclass (.data-table.my-table) for min-width overrides only.

Chips and Badges

For chip borders, use --color-accent-border or the color-mix() approach for semantic colours — not the full-strength text token.

❌ Don't
border: 1px solid var(--color-info);
border: 1px solid var(--color-purple);
✅ Do
border: 1px solid var(--color-accent-border);
border: 1px solid color-mix(
  in srgb,
  var(--color-info) 35%,
  transparent
);

Status and Feedback

Never use alert(), confirm(), or prompt(). Use the helpers in global.js:

// Transient notifications
showToast('Saved successfully.');
showToast('Failed to save.', 'error');
showToast('Processing…', 'info', 8000); // custom duration ms

// Destructive confirmation — returns Promise<boolean>
if (!await showConfirm('Delete this entry?', 'Delete')) return;
if (!await showConfirm('Delete entry?', 'Delete', 'Confirm Deletion')) return; // custom title

// Inline field validation
setFieldError(document.getElementById('name-input'), 'Name is required.');
clearFieldError(document.getElementById('name-input'));
clearAllFieldErrors(document.getElementById('my-form'));

Icon System

All icons use Tabler Icons (outline set): viewBox="0 0 24 24", stroke-width="2", round caps/joins, fill="none", MIT licensed. They ship as a single sprite at static/icons.svg, each a <symbol id="icon-{slug}">.

<!-- In templates: reference by <use> -->
<svg class="svg-icon" width="14" height="14" aria-hidden="true"><use href="/icons.svg#icon-device-floppy"/></svg>
// In JS: use the svgIcon(name, size = 14) helper from global.js
btn.replaceChildren(svgIcon('pencil'), document.createTextNode(' Edit'));   // icon + label
delBtn.setAttribute('aria-label', 'Delete'); delBtn.appendChild(svgIcon('trash')); // icon-only
  • .svg-icon (vertical-align: middle) aligns the icon with adjacent text in any inline context; the svgIcon() helper adds it automatically.
  • Icons in headings: add class="icon-heading" to the heading — it flex-centres the fixed-size icon on the heading's optical centre (plain vertical-align sits it low next to large text).
  • Icons inherit colour via currentColor — they adapt to text colour and theme automatically.
  • Standalone pages not built on layout.html (e.g. login.html) must include <script src="/global.js"> before their page script to get svgIcon.
  • Adding an icon: copy the paths from github.com/tabler/tabler-icons/.../outline/{slug}.svg into a new <symbol>.
Emoji policy. No emoji in UI chrome — headings, tabs, buttons, stat cards, badges, illustrations, and category labels all use SVG icons. See Approved Exceptions for the narrow permitted set (copy-to-game text, user-entered icon fields, plain typographic glyphs, <option> labels).

Dark Mode Rules

  1. Always use tokens. Every colour must come from a var(--color-*) token.
  2. Use html[data-theme="dark"] for overrides, not html.theme-dark.
  3. Include both selectors for color-scheme (see Theme System above).
  4. Use color-mix() for semi-transparent semantic borders — it adapts automatically as the token changes between themes.
  5. Test in both themes before committing.
/* Semi-transparent semantic border — adapts to both themes */
border: 1px solid color-mix(in srgb, var(--color-danger) 35%, transparent);

/* Chart.js — read token at runtime */
Chart.defaults.color = getComputedStyle(document.documentElement)
    .getPropertyValue('--color-text-mid').trim();

JavaScript DOM Standards

Safe DOM construction

❌ Never
container.innerHTML =
  `<div class="card">
    ${item.name}
  </div>`;
✅ Always
const card = document.createElement('div');
card.className = 'card';
card.textContent = item.name;
container.appendChild(card);
// Batch replace children cleanly
container.replaceChildren(...items.map(buildCard));

Event binding

❌ Never
btn.setAttribute('onclick',
  `deleteItem(${item.id})`);
✅ Always
btn.addEventListener('click',
  () => deleteItem(item.id));

Colours in JS

❌ Never
span.style.color = '#1d4ed8';
el.style.cssText =
  'color:#f87171;font-size:0.85em;';
✅ Always
span.style.color = 'var(--color-info)';
el.style.cssText =
  'color:var(--color-danger);font-size:0.85em;';

If the same styled element is created in a loop, the style belongs in a CSS class — not duplicated in each JS call.

el() helper

For files that construct significant DOM, use the builder helper from dashboard.js rather than chained createElement calls. Copy the el(tag, props, ...children) function verbatim — it's the reference pattern for new files.

Approved Exceptions

Emoji & non-icon glyphs

UI chrome uses SVG icons (see Icon System). A literal glyph or emoji is permitted only here:

  • Copy-to-game text output — Desert Storm battle mail, Schedule Discord/chat text, the schedule canvas PNG. SVG can't render in plaintext, and the marker is part of the copied content (🏜️ ⏰ 💪 ⚡).
  • User-entered icon fields — Schedule event-type icon inputs (et-icon, se-icon) and the season-template icon cell store a user-chosen emoji as data; placeholders (📅 🌐 🎯) hint an emoji is expected.
  • Plain-text typographic symbols — ✓ ✗ ★ ✕ — used inline in compact status text / column headers render consistently and are tolerated. Prefer svgIcon('check') / svgIcon('x') for prominent status.
  • <option> labels — SVG can't live in <option>; squad-type and upload-category selects use plain text (the icon shows on the resulting card/badge).

Hardcoded colours

The following hardcoded colour values are intentional and must not be replaced with tokens.

Seat tier dots (recruiting.css)

.seat-gold   { background: #f1c40f; }
.seat-purple { background: #9b59b6; }
.seat-blue   { background: #3498db; }
.seat-grey   { background: #95a5a6; }

These map to specific in-game seat tier colours that must be constant across both themes. They are game identifiers, not semantic states. New in-game colour coding that requires constant values must be documented here.

Sidebar shell background

--color-sidebar is a near-black value in both light and dark themes — the sidebar is intentionally always dark regardless of page theme.
